# Security Policy

## Supported Versions
- v1.0.0 (macOS binary bundle)

Older artifacts are not actively maintained. Use the latest release for security fixes.

## Reporting a Vulnerability
- Please report privately via GitHub Security Advisories (Security tab) instead of opening a public issue.
- Include: affected version, macOS version, reproduction steps, and any logs or screenshots that help clarify impact.
- If the issue involves sensitive information or requires coordination, note that in your report so we can arrange a secure channel.

## Response
- We aim to acknowledge new security reports within 5 business days.
- If a fix is required, we will coordinate disclosure timing and provide mitigation guidance in the release notes and changelog.

## Safe Use
- Use binaries from the official releases page and verify checksums before execution.
- Avoid running unofficial or repackaged builds.
